## Design Context

### Users
- Uso pessoal do desenvolvedor, evoluindo para multi-user com componente social (lista de amigos, perfis publicos)
- Dois contextos: planejamento em casa (desktop/mobile) e execucao no treino (mobile, uma mao, entre series)
- Possibilidade de terceiros criarem planos para o usuario
- Job to be done: montar treinos, registrar sets/cargas em tempo real, acompanhar evolucao, compartilhar progresso

### Brand Personality
- **Moderno**: visual contemporaneo, nao generico
- **Energetico**: transmitir motivacao e intensidade sem ser agressivo
- **Limpo**: informacao essencial, zero ruido visual, organizado e com espaco para respirar

### Aesthetic Direction
- Visual tone: clean com acentos vibrantes — superficies neutras com cor nos pontos de acao
- Cores base: indigo (#6366f1 / #818cf8) como primaria, amber (#d97706 / #fbbf24) como accent, charcoal neutro como base
- Dark mode como default (contexto academia com telas escuras), light mode disponivel
- Anti-reference: nao parecer dashboard corporativo, nao parecer app de saude/wellness pastel, nao usar ilustracoes ou mascotes, NAO parecer interface gerada por AI (sem glows neon, sem gradientes decorativos sutis, sem hover glow generico em todos os cards, sem paleta lavender/purple generica)
- Referencia propria: seguir o design system ja existente no projeto
- Tipografia: Inter (corpo) + Space Grotesk (display/numeros) + JetBrains Mono (dados). Pesos contrastantes (bold para numeros/metricas, light para labels)
- Espacamento generoso — touch targets grandes para uso no treino

### Design Principles
1. **Mobile-first, gym-first**: toda decisao de layout parte do cenario "celular na mao entre series". Botoes grandes, inputs acessiveis, informacao escaneavel
2. **Data over decoration**: numeros (peso, reps, series) sao protagonistas. Hierarquia visual favorece metricas
3. **Minimal friction**: minimo de taps para registrar um set. Fluxo de treino ativo e a prioridade #1
4. **Hierarquia por espaco**: usar spacing e agrupamento para comunicar estrutura, nao bordas e cards excessivos. Menos containers, mais ritmo visual
5. **Energy through contrast**: usar cor e peso tipografico para criar energia, nao ilustracoes ou animacoes excessivas
6. **Progressive disclosure**: mostrar o essencial, revelar detalhes sob demanda (expandir historico, notas, etc.)
7. **No AI slop**: zero efeitos decorativos sem proposito (glows, pulses infinitos, gradientes cosmeticos). Cada efeito visual deve comunicar estado ou guiar acao

### Tech Constraints
- React 19 + TypeScript + Vite
- CSS puro com custom properties (sem Tailwind, sem CSS-in-JS)
- FontAwesome via CDN (solid + brands)
- Recharts para graficos de evolucao
- Sem component library (UI construida do zero)
- Layout mobile-first, single-column, max-width 480px

### Navigation Architecture
- Bottom nav com 4 itens: Home, Treino, Progresso, Conta
- "Conta" unifica Perfil + Configuracoes numa unica tab
- Treino na nav redireciona para sessao ativa quando existe uma

### Accessibility
- Contraste minimo garantido entre texto e fundo
- Touch targets minimos de 44px (ja implementado nos botoes)
- Sem requisitos formais de WCAG, mas boas praticas basicas
- Labels visiveis em todos os formularios (nao depender de placeholders)
- Focus management em modais e dialogs
